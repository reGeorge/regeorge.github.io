# 书籍碰撞流程 (Book Collision Protocol)

本文件定义了将书籍与个人哲学洞见库进行碰撞、并将有效碰撞升级为洞见卡的完整流程。

---

## 🎯 触发场景

当用户发送以下任意一种输入时，启动本流程：

- 书名 + 读后感受（哪怕只有一句话）
- 书名 + 摘录片段
- 书名 + "帮我碰撞"

**不要**在用户只是提到书名但未表达任何感受或想法时主动启动碰撞。
**不要**在用户明确说"只是记录读过"时生成碰撞分析。

---

## 📐 流程结构

### Step 1：生成书籍快照卡

读取用户输入，提炼以下三项，**不得超过这三项**：

- **核心主张**：这本书最想说的一件事（一句话）
- **预设读者**：这本书默认你是谁、有什么问题
- **与你的距离**：这本书的出发点和你的认知起点，是接近还是有张力

快照卡写入：
`~/Documents/codeStore/obsidian/阅读/{日期}-{书名}.md`

格式如下：

```
---
书名：{书名}
作者：{作者}
读完日期：{日期}
状态：已读 / 在读 / 放弃
---

## 核心主张
{一句话}

## 预设读者
{一句话}

## 与我的距离
{一句话，接近 / 互补 / 张力}

## 原始感受（用户原话）
{保留用户原话，不做修饰}
```

---

### Step 2：碰撞扫描

将书籍快照与用户现有哲学洞见库进行对比，输出三类碰撞：

#### A. 强共鸣点
你已有这个认知，书给了更清晰的语言或更坚实的论据。
→ 输出：已有哪张洞见卡 + 书的哪个表述可以加固它

#### B. 互补点
你和书各占一半，单独都不完整，合并后产生新的认知。
→ 输出：你的认知边界在哪 + 书补充了什么 + 合并后的新陈述是什么

#### C. 真正的张力点
你的认知与书存在实质分歧，不是表达不同，而是逻辑方向相反。
→ 输出：分歧的核心命题是什么 + 你的立场 + 书的立场 + 你是否愿意修正

**严格禁止**：
- 不准把表达风格差异误判为张力
- 不准为了输出"张力"而强行制造对立
- 不准在用户未提供洞见库内容时凭空捏造碰撞

---

### Step 3：人工裁决门禁（Human-in-the-loop）

碰撞扫描完成后，**停下来**，向用户呈现：

```
碰撞扫描完成。

【强共鸣】{N} 个
【互补点】{N} 个
【张力点】{N} 个

哪些你想升级为洞见卡？
哪些只是"读过，知道了"？
哪些你想反驳？
```

**不要**在用户确认前自动生成洞见卡。
**不要**把所有碰撞点都推荐升级，默认只有互补点和张力点值得升级。

---

### Step 4：升级为洞见卡（按需执行）

仅对用户确认的碰撞点执行。

读取 `INSIGHT-CARD-TEMPLATE.md` 获取卡片结构约束，生成洞见卡。

升级时的额外约束：

- **来源字段**必须同时标注书籍来源和你的原始洞见来源
- **冗余过滤器**中的噪音，优先使用书中被你过滤掉的表述（而非泛泛陈词）
- **对话还原**至少有一组 Q&A 体现"书的视角 vs 你的视角"的交锋
- **哲学笔记**模型名称格式：`{书名} × {你的洞见标题}`，体现碰撞来源

写入路径遵循 `INSIGHT-CARD-TEMPLATE.md` 中的落地规则。

---

### Step 5：精准推荐（按需执行）

触发条件：用户主动问"推荐我读什么" **或** 碰撞完成后用户说"还有什么适合我读"。

推荐逻辑：

1. 扫描现有洞见库，找出**认知空白维度**——有感受但缺语言、有直觉但缺框架的区域
2. 从已读书籍快照卡中找互补书（同一维度的不同角度）
3. 从未读书籍中推荐，必须说明：这本书填你哪个空白，而不是"你可能喜欢"

推荐格式：

```
你在"{洞见维度}"上有强烈的个人认知，但缺少{框架/语言/反例}。

推荐：《{书名}》
理由：它的核心主张是"{一句话}"，
      恰好从{角度}切入你已有的"{洞见卡标题}"。
```

**严格禁止**：
- 不准基于书籍类型或风格相似性推荐
- 不准一次推荐超过 3 本
- 不准在没有扫描洞见库的前提下给出推荐

## 🚫 全局禁止

1. 不准在用户未确认前自动执行 Step 4
2. 不准把书的观点直接写入洞见卡，必须经过碰撞、有你的原始认知作为锚点
3. 不准用书的语言覆盖你的原话，原话永远优先
4. 不准生成"这本书很好，值得一读"类型的书评式总结
5. 不准把"读过"等同于"碰撞完成"

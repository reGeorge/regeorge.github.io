# 数据飞轮系统搭建指南

给另一只虾的学习文档。核心思路：用本地 git 替代远程部署，所有数据存本地。

---

## 一、系统总览

```
~/data-flywheel/
├── .rules/                    # 系统规则（AI 的行为约束）
│   ├── book-collision-protocol.md   # 书籍碰撞协议
│   ├── insight-card-template.md     # 洞见卡模板
│   └── obsidian-writing-rules.md    # Obsidian 写入规则
├── obsidian/                  # Obsidian 笔记库（本地 git）
│   ├── 阅读/                  # 书籍快照卡
│   ├── 灵感/                  # 灵感与思考
│   ├── 生活/                  # 生活记录
│   └── ...
└── flywheel/                  # 飞轮展示页面（本地 git）
    ├── index.html             # 飞轮主页
    ├── sources/               # 碰撞来源文件
    └── ...
```

**两个 git 仓库：**
- `obsidian/` → 个人笔记库，平时用 Obsidian 打开
- `flywheel/` → 飞轮展示页面（纯静态 HTML）

## 二、核心规则文件

### 2.1 书籍碰撞协议 (.rules/book-collision-protocol.md)

这是整个飞轮最核心的流程，定义了"读书 → 碰撞 → 升级"的完整链路。

```
触发条件：用户发送 书名 + 读后感受（至少一句话）

Step 1：生成书籍快照卡
- 提炼三项：核心主张（一句话）、预设读者、与你的距离
- 写入 obsidian/阅读/{日期}-{书名}.md
- 格式：
  ---
  书名：xxx
  作者：xxx
  读完日期：2026-xx-xx
  状态：已读
  ---
  ## 核心主张
  ## 预设读者
  ## 与我的距离

Step 2：碰撞扫描
将书与现有洞见库对比，输出三类：
A. 强共鸣点：你已有这个认知，书给了更好的语言/论据
B. 互补点：你和书各占一半，合并后产生新认知
C. 张力点：存在实质分歧（不是表达风格差异）

严格禁止：
- 不准把表达风格差异误判为张力
- 不准为了输出张力而强行制造对立

Step 3：人工裁决（关键！）
碰撞扫描完成后必须停下来，向用户展示结果，等待确认。
默认只有互补点和张力点值得升级。
不要在用户确认前自动生成洞见卡。

Step 4：升级为洞见卡（仅对用户确认的执行）
读取洞见卡模板，按格式生成。

Step 5：精准推荐（仅在用户主动问时触发）
基于洞见库找认知空白，推荐不超过3本，说明填哪个空白。
```

### 2.2 洞见卡模板 (.rules/insight-card-template.md)

每张洞见卡是 HTML 格式的折叠卡片，结构如下：

```html
<div class="insight-card" style="border-color: var(--accent); background: rgba(217, 119, 87, 0.02);">
    <!-- 卡片头部：标签 + 书名徽章 -->
    <div class="card-header" style="display:flex;align-items:center;gap:8px;flex-wrap:wrap;">
        <span class="tag" style="margin-bottom:0;">{类型}：{一句话核心}</span>
        <span class="book-badge">📖 {书名}</span>
    </div>

    <!-- 折叠区域：展开看推导过程 -->
    <details>
        <summary data-hint="（点击展开推导过程）">
            <span class="core-text">{碰撞结论，3-5句话}</span>
        </summary>
        <hr class="divider">
        <div class="process-content">
            <!-- 核心淬炼：一句话真知 + 类比 -->
            <div class="process-section">
                <h4>核心淬炼</h4>
                <p><strong>一句话真知：</strong>{最核心的洞见}</p>
                <p><strong>类比：</strong>{用一个具象比喻解释}</p>
            </div>

            <!-- 对话还原：Q&A 形式，书的视角 vs 你的视角 -->
            <div class="process-section">
                <h4>对话还原</h4>
                <div class="dialogue-block">
                    <span class="q">书：{书的核心表述}</span><br>
                    <span class="a">你的回应：{你的洞见}</span><br><br>
                    <span class="q">追问：{深入问题}</span><br>
                    <span class="a">{深入回应}</span>
                </div>
            </div>

            <!-- 身体感悟：真实的身体层面的体验 -->
            <div class="process-section">
                <h4>身体感悟</h4>
                <div class="aha-block">
                    <p>{具体的身体/情绪体验，不是抽象描述}</p>
                </div>
            </div>

            <!-- 冗余过滤器：已过滤的噪音 + 聚焦透镜 -->
            <div class="process-section">
                <h4>冗余过滤器</h4>
                <div class="redundancy-section">
                    <h4>已过滤的噪音</h4>
                    <span class="noise">{噪音1}</span>
                    <span class="noise">{噪音2}</span>
                    <span class="noise">{噪音3}</span>
                    <div class="focus-lens">
                        <strong>聚焦透镜：</strong>{为什么以上是噪音，真正的洞见在哪}
                    </div>
                </div>
            </div>

            <!-- 最小化行动实验 -->
            <div class="process-section actionable-section">
                <h4>最小化行动实验</h4>
                <div class="action-item">
                    <div class="checkbox"></div>
                    <span><span class="action-label">{今天的实验}：</span>{具体操作}</span>
                </div>
            </div>

            <!-- 可迁移场景 -->
            <div class="process-section migration-section">
                <h4>可迁移场景</h4>
                <div class="migration-item">{场景1：具体描述}</div>
                <div class="migration-item">{场景2：具体描述}</div>
            </div>
        </div>
    </details>

    <!-- 哲学笔记：卡片折叠状态下的底线信息 -->
    <div class="philosophy-note">
        <strong>{模型名}：</strong>{一句话总结}
    </div>

    <!-- 来源链接 -->
    <div class="source-link">
        来源：<a href="sources/{碰撞来源文件}">[{来源描述}]</a>
    </div>
</div>
```

### 2.3 Obsidian 写入规则 (.rules/obsidian-writing-rules.md)

```
规则1：双重锚定
- 来源锚定：必须有明确的来源文件或对话历史
- 原话锚定：必须引用用户原话，不偏离不捏造

规则2：负面约束
- 不准在没有 .md 文件或对话记录的情况下自行联想细节
- 不准用 AI 生成的虚构案例填充内容
- 不准忽略"贴近原话"的要求

规则3：书籍快照卡追加碰撞记录
碰撞完成后，在快照卡末尾追加：
---
## 碰撞记录
**扫描日期：** {日期}
### 强共鸣 / 互补点 / 张力点
- {编号}：{碰撞描述} → **已升级** / 读过不升级
### 张力点裁决
- {用户裁决内容}
### 升级后的洞见卡
{编号列表}
```

## 四、本地 Git 设置

既然另一只虾访问不了外网，用本地 git 管理数据：

```bash
# 初始化
mkdir -p ~/data-flywheel/{.rules,obsidian,flywheel}
cd ~/data-flywheel && git init

# 首次提交
git add -A && git commit -m "init: data flywheel system"

# 日常使用：每次更新后
git add -A && git commit -m "update: {描述本次更新}"
```

如果之后能联网，可以推到 GitHub：
```bash
git remote add origin git@github.com:{user}/{repo}.git
git push -u origin main
```

## 五、工作流程总结

```
用户输入（微信/对话）
    │
    ├─ 原始洞见 → 直接生成洞见卡 → 写入 flywheel/index.html + flywheel/sources/
    │                                    同时在 obsidian/ 记录原始笔记
    │
    ├─ 读书感受 → Step1 生成快照卡 → Step2 碰撞扫描
    │                                   → Step3 等待用户确认
    │                                   → Step4 生成碰撞洞见卡
    │
    └─ 日常记录 → 写入 obsidian/ 对应目录
                    （阅读/生活/工作/灵感/...）

每次操作后：git add -A && git commit
```

## 六、核心设计理念

1. **双重锚定**：每张卡必须有来源文件和用户原话，不准凭空捏造
2. **人工裁决**：碰撞扫描后必须等用户确认，不自动升级
3. **密度优先**：删到不能再删，留下的每句话都携带信息量
4. **指向自己**：所有碰撞的产出必须落在用户自己的认知上，不是书评
5. **可操作**：每张卡必须有"最小化行动实验"和"可迁移场景"

# 莉莉丝的可爱角色卡片页 PRD v1

## 1. 背景

当前页面使用手写 SVG 实现花园、池塘、小狗钓鱼等复杂场景。这个方案维护成本高，角色细节难画好，最终视觉效果也不够稳定。

新的方向改为一个静态展示页面：用漂亮、柔和、适合小朋友的卡片展示可爱角色。页面不追求复杂互动，优先追求好看、清晰、易维护。

## 2. 目标

- 做一个小朋友喜欢看的静态卡片页面。
- 页面第一屏就能看到可爱的角色卡片，而不是复杂说明。
- 视觉风格温暖、明亮、圆润、有童趣。
- 后续可以很容易替换角色、增加卡片、换图片。
- 避免直接使用有版权风险的官方动漫角色图片。

## 3. 非目标

- 不做复杂小游戏。
- 不继续维护大型手写 SVG 场景。
- 不直接使用官方星之卡比、皮卡丘、迪士尼等角色素材作为公开页面主图。
- 不做需要后端或构建工具的功能。

## 4. 用户

主要用户是小朋友和陪她一起看的大人。

页面应该让小朋友第一眼觉得“可爱、想点开、想继续看”，让大人觉得“安全、简单、好维护”。

## 5. 页面内容

### 5.1 页面标题区

- 标题：莉莉丝的可爱角色收藏
- 副标题：今天想和哪个小伙伴一起玩？
- 保留返回首页入口。

标题区不做大段说明，避免页面像文档或产品介绍。

### 5.2 角色卡片区

默认展示 4 张角色卡片。

每张卡片包含：

- 角色图片
- 角色名字
- 一句短描述
- 一个小标签，例如“勇敢”“爱笑”“会唱歌”“喜欢星星”

示例角色方向：

- 粉粉小星：圆圆软软，喜欢在云朵上跳舞。
- 蓝莓飞飞：带着小翅膀，最喜欢发现新朋友。
- 奶油小熊：温柔慢慢走，口袋里总有糖果。
- 彩虹兔兔：每天收集一种新的颜色。

这些角色应是原创角色，不直接使用官方 IP 名称或官方形象。

### 5.3 可选氛围元素

可以加入少量轻量装饰：

- 星星
- 云朵
- 彩虹
- 花朵
- 小贴纸形状

这些装饰用 CSS 或简单 SVG 即可，不再绘制复杂完整场景。

### 5.4 背景音乐

保留一个“播放音乐 / 暂停音乐”按钮。

音乐可以继续使用浏览器 Web Audio 生成的轻快小旋律，避免引入第三方音乐版权问题。

要求：

- 默认不自动播放。
- 用户点击后才播放。
- 音量柔和，不刺耳。
- 可以随时暂停。

## 6. 图片来源与版权策略

页面如果公开部署到个人网站，应优先使用低风险素材。

允许的素材来源：

- 自己画的原创角色。
- 小朋友自己的画，经处理后作为卡片图片。
- AI 生成的原创角色，但提示词不能要求生成“星之卡比”“皮卡丘”等受保护角色。
- CC0 或 Public Domain 图片。
- CC BY 图片，但页面需要按授权要求署名。

不建议使用：

- 官方动漫角色图、游戏截图、海报、Logo。
- 网上随手下载的粉丝图或同人图。
- AI 生成的明确仿官方 IP 角色图。

设计原则：

- 可以借鉴“圆润、梦幻、粉色、星星、可爱冒险”等儿童审美。
- 不直接复制具体角色的名字、造型、配色组合和标志性元素。
- 如果未来要使用真实 IP 角色，只作为本地私人页面更合适；公开页面需要谨慎处理。

## 7. 视觉风格

整体关键词：

- 明亮
- 柔软
- 糖果色
- 干净
- 适合小朋友

布局要求：

- 桌面端：2 到 4 列卡片网格。
- 移动端：单列卡片，图片和文字都要清楚。
- 卡片圆角适中，不要过度堆叠卡片。
- 字体要大、清楚、容易读。
- 避免页面被单一粉色占满，可搭配蓝、黄、绿、紫等轻柔颜色。

动效要求：

- 卡片 hover 时轻轻上浮。
- 小星星或贴纸可以慢慢闪动。
- 不做大量连续动画，避免分散注意力。

## 8. 技术方案

使用纯静态实现：

- HTML
- CSS
- 少量原生 JavaScript

图片资源可以放在：

- `projects/gardenForLilis/assets/`

首版如果没有真实图片，可以先用 CSS 绘制的简单原创角色占位，或使用内联 SVG 小头像作为临时方案。

不引入框架，不引入构建流程。

## 9. 首版验收标准

- 打开页面后，第一屏能看到标题和至少 4 张可爱角色卡片。
- 页面不再依赖复杂大型 SVG 场景。
- 每张卡片有角色图、名字、短描述和标签。
- 页面在手机宽度下不横向溢出。
- 音乐按钮可播放和暂停背景音乐。
- 页面素材没有直接使用官方动漫 IP 图片。
- 代码结构简单，后续新增角色只需要复制一张卡片或改一份角色数据。

## 10. 后续扩展

- 增加“今天随机抽一个小伙伴”按钮。
- 增加按颜色或性格筛选角色。
- 增加小朋友自己的画作上传或替换入口。
- 为每个角色增加一句语音或小故事。

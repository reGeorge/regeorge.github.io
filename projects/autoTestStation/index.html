Part 01
从 caseHelper skills 到工作站界面：4 步把需求落到可汇报的事实
入口收编 → 结构化沉淀 → ATP 快照刷新 → 一键导出报告。每一步都在 meta.json / board_state.json 留痕，下一步直接在已有事实基础上继续。

入口收编

workstream 目录、meta.json、主 CSV、关联文档、ATP taskId 都在新增需求时定下来。改名、迁版本、取消托管全部走 /api/workstream/*，不动文件系统直连。

Versioned Workspace
meta.json + board_state
🧱

结构化沉淀

用例行、生命周期字段、ATP 绑定、CI 状态、阶段说明都折叠到同一个 workstream 对象，多个 tab 共享同一份事实，不需要为每个面板单独维护数据。

cases / lifecycle / atp
single source of truth
🔁

ATP 快照

调 /case/v2 取 pathTree，按 rootCaseId 下 1 层子目录归桶；同 taskId 的多个需求一次刷新一起更新，避免逐个点。scope_summary 与 task 全量并存。

scope aggregation
cascade refresh

右侧总结报告，一键导出可粘贴的表格

把跨需求的 ATP 通过率聚合成一张表，按通过率升序排，备注和负责人就地编辑。一键复制提供富文本表格、Markdown、纯文本三种格式，直接粘到飞书或邮件。
